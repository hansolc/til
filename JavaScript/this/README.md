# This
<hr>

자신이 속한 객체 혹은 자신이 생성 할 인스턴스를 가리키는 `자기참조변수`이다.

### this가 필요한 이유
- 리터럴 객체에서는 메서드 내부에서 자신의 프로퍼티를 접근할 수 있어야 한다.
- 생성자 함수를 통해 인스턴스를 생성하는 경우, 인스턴스를 생성하는 명령어를 입력하기 전에 생성자 함수가 존재해야한다. 그때 생성자 함수는 메서드와 프로퍼티를 추가하기 위해 자신이 생성할 인스턴스를 참조할 수 있어야 한다.

### this의 바인딩
this는 함수 호출 방식에 의해 결정된다.
1. 리터럴 객체 => 자기자신
2. 생성자 함수 => 자신이 생성할 인스턴스
3. 일반함수 혹은 전역 => window
4. use strict 모드 => undefined
`ex1.js`

this는 자신의 프로퍼티 혹은 메서드에 접근하기 위한 변수로 일반함수 에서는 this가 거의 필요없다. 따라서 use strict모드를 통해 undefined와 바인딩 가능하다.

#### this의 명시적 바인딩
일반 함수에서는 this는 전역객체에 바인딩 되기 때문에 리터럴객체 혹은 생성자 함수내의 메서드에서 중첩함수 혹은 콜백 함수를 사용할 경우 메서드 내에서는 전역객체를 바인딩 한다. 이런 경우의 함수는 자신이 생성할 인스턴스의 변화를 주기 위해 사용되기 때문에 this의 명시적 바인딩이 필요하다.

1. 변수를 통해 바인딩
2. prototype.[apply/call/bind] 을 통해 바인딩
3. 화살표 함수 사용
`ex2.js`
